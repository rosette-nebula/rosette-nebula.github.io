变量提升和函数提升

这里只讨论var创建的变量，不讨论ES6中const和let。 变量提升和函数提升: 其实是变量和函数都会在编译阶段被放入到内存中，而在代码中的位置不变。

变量提升(Hoisting)

  • 只会提升变量的声明，不会提升初始化

  • 变量提升时，如果只声明没有初始化，那么变量被使用时会自动赋值为undefined

  • 声明变量并初始化,再打印结果

var num = 6;
console.log(num); // 输出6

函数比变量优先提升

var num;
console.log('1---', num); //  ƒ num() { console.log('函数'); }
function num() {
  console.log('函数');
}
console.log('2---', num); //  ƒ num() { console.log('函数'); }
num = 6;
console.log('3---', num); // 6

总结

由此看来，变量与函数都会提升，但是会有差别。

  1. js 会扫描各个作用域中的函数，并将它们存入到内存中(优先提升到各个作用域的顶部)。
  2. js 会扫描各个作用域中的变量，并且判断它们是否被使用和赋值。
  • 当已存在同名函数时，未运行到变量赋值语句时，一直使用同名函数。当运行到变量赋值语句时，就将变量提升并赋值。
  • 当不存在同名函数时，则放入到内存中(变量提升)，并赋值为undefined
